Identify the Linux Design Philosophy

One of the first operating systems still in use is Unix. It was developed in 1969 but was never made available as open-source software. Instead, a variety of tech companies, such as IBM, Hewlett-Packard, and AT&T, were linked to Unix versions. The multiple Unix "flavours" that each corporation has exclusive rights to use are referred to as Unix versions.

Two of the three main operating systems available today (Linux, macOS, and Windows) have their origins in Unix. The Linux kernel and many other standards are shared by the macOS kernel, which was developed from a Unix variant called BSD (Berkeley Software Distribution). Apple's OS is not, however, a FOSS (free and open-source software). Another proprietary kernel with a more stringent licensing scheme is used by Microsoft Windows.

Traits of Linux OS

  • Free: There is no licensing fee or anything with Linux.
  • Support: There is no community-driven support with Linux.
  • Security: Because of the open-source nature of Linux and its associated software, many developers can and do review code for vulnerabilities.
  • Performance: Linux provides better performance when compared to other operating systems.
  • Software availability: Fewer or less familiar software options may exist, especially for nonbusiness applications, such as games.
  • Learning: Some feel Linux is a bit difficult as compared to Windows and Mac.

Linux Distribution

Due to the fact that anyone can develop and distribute their own version of Linux, there are countless alternatives. Distributions (or "distros" for short) are the collective term for these distinct releases. Distributions are specialised Linux releases that focus on a particular issue, like system security or application hosting.

Some of the most common distros include:

  • Fedora Linux
  • Ubuntu Desktop, Server, Core
  • Red Hat Enterprise Linux (RHEL)
  • Linux Mint

The Command-Line Interface

One of the amazing features of Linux compared to other OS is its CLI. Linux administrators on a daily basis use its CLI for daily tasks.

On comparing to the GUI, CLI has advantages like:

  • Quicker: It's usually fast as commands can be executed faster.
  • Performance: CLI consumes little or lesser hardware.
  • Scriptable: CLI commands can be written in a text file and can be used to execute a series of scripts or commands.

Enter Shell Commands

A piece of software called a shell provides the CLI. The user's input is accepted by the shell, which checks it for syntax before returning the user's output. Bash is the default shell for the majority of Linux distributions, thus sysadmins should be familiar with it.

Bash Syntax

When entering the command, it should be in a specific structure and syntax which comprises of:

  • Command: Primary instruction is given to the system.
  • Sub Command: A secondary instruction covering the primary command.
  • Option: It is a type of command modifier that can change the output of the command.
  • Argument: It is the object on which action is being made.

Normal Syntax

The three main parts of the Bash syntax are used in the standard command syntax: the command, options to change the command, and an argument for the command to operate on. As an example, we can see below several ways to use the ls command.

ls: This command is used to list the directory and files present in the system. By default, when you run the ls command it will take the present working directory.

ls -la: List all content in a long format where ls stands for list, -a stands for all and shows hidden files and directories, -l stands for long and displays the files and directories in a detailed format, including file permissions, ownership, size, and modification time.

ls /var/log: List the content of the specified directory.

Command Sub Command

Many Linux commands allow the use of subcommands to define the precise data that a sysadmin requires. The core structure of typical command syntax is not used for these commands. The system administrator types the principal command, a space, and a subcommand, followed by an argument.

Below we can see the example of the ip command:

ip addr: Display all the IP addresses for all interfaces.

ip addr show eth0: Shows information about eth0 only.

ip help: Display basic help about the ip command.

Cat Command

The cat command is a commonly used command in Bash to read, concatenate and display the contents of one or more files. The basic syntax for the cat command is:

cat {filename.txt}

Basic Bash Commands

There are lots of commands in Bash. Some of the most often used commands have to do with managing files, such as seeing files and their contents, changing directories (or folders), or modifying files.

Command Usage
ls List the directories
touch Create a new empty file
cat Display the content of a file
cd Change the directory
less Display the content of the file that fits on the screen
shutdown As the name suggests, shutdown the system

Get Help with Linux

In Linux, there are numerous ways to receive assistance. The manual pages, or "man pages" for short, are the most prevalent. The system and some apps both have built-in documentation. There are other online resources as well, and they are frequently the most recent. It's fairly typical to utilize the man pages as a quick reference for presenting the available options because there are so many commands and options for each command.

When working as a new Linux user one can get stuck but can use the help of man as man pages are the primary reference for standard Linux commands. The man pages provide syntax information and usage examples. Perhaps most important, the available options are displayed.

The syntax for using the man command is:

man {command}

Man Page Navigation

  • Home -- Move to the beginning of the man page.
  • End -- Move to the end of the man page.
  • Page Up -- Scroll up one page.
  • Page Down -- Scroll down one page.
  • / -- Begin a search for a term or text string.
  • n -- Move to the next occurrence of the search term.
  • p -- Move to the previous occurrence of the search term.
  • q -- Exit the man page, and return to the shell.
